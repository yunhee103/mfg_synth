# 탐색 워크시트

작성 규칙: 스크립트 출력을 보고 빈칸을 채운다. 판별이 안 되는 항목은
'판별 불가'라고 쓰고 넘어간다 (나중에 같이 판별한다).

## 1. 테이블 인벤토리 (A 출력 기준)

| 테이블 | 행 수   | 이 테이블은 무엇에 대한 기록인가 (한 줄)     |
|---|-------|------------------------------|
| customers | 30    | 고객사                          |
| suppliers | 15    | 공급사                          |
| items | 180   | 완제품과 자재를 다 담은 품목 마스터         |
| bom | 361   | 완제품별 자재 구성표                        |
| sales_order_h | 2048  | 주문 헤더 (주문 1건당 1행, 고객·주문일)    |
| sales_order_d | 3075  | 주문 상세/명세 (품목 1개당 1행, 수량·납기)  |
| purchase_orders | 2465  |          구매 목록       |
| material_receipts | 2431  |       자재 입고              |
| shipments | 3002  |      제품 출하       |
| inventory_daily | 36960 | 일별 자재 재고 현황 (매일 자재별 보유,발주잔량) |
| item_change_log | 8    |     품목 변경 로그        |

## 2. 관계도 (B 출력 기준)
  수주상세 -> 수주헤더 고아: 0
  수주상세 -> 품목마스터 고아: 0
  출하 -> 수주상세 고아: 0
  입고 -> 발주 고아: 0
  BOM 자식 -> 품목마스터 고아: 0
  수주상세 중 미출하 건수: 73

B에서 0이 아니었던 체크: 수주상세 중 미출하 73건 (나머지 관계 5개는 전부 0=정상)
그것이 이상한가, 자연스러운가: 판별 보류 — 2.4% 규모라 12월 꼬리로 추정되나 due_date 분포 확인 전까지 결론 유보 (Q3로 이관)

## 3. 위화감 목록

형식: 현상 / 어디서 봤나 / 왜 이상한가
1. 위화감 : 
현상 - ERP 코드(F-0001)와 MES 코드(F0001)가 하이픈 유무로 다름. 
       게다가 MES엔 CONN-39 같은 ERP에 없는 코드도 섞여 있음.
어디서 - explore.py C-1 출력, "일치 0/70종"
왜 이상 - 같은 완제품인데 코드가 안 맞아서 두 시스템을 직접 조인하면 하나도 안 이어짐. 통합하려면 코드 변환이 먼저 필요함.
2.  위화감 :
현상 - MES 양품수량과 ERP 출하수량이 대부분(991건 중 970건) 일치하는데, 21건(약 2%)만 안 맞음.
어디서 - explore.py C-2, "불일치 21건 / 991건"
왜 이상 - 만든 개수와 내보낸 개수는 같아야 정상인데, 소수 건에서만 
         어긋남. 왜 이 21건만 다른지 확인 필요.
3. 위화감 :
현상 - 검사 Excel 검사일이 다양한 형식으로 섞임(20250502, 5/2, 25.5.7). 
       검사자마다 형식이 고정된 걸 보니 수기 입력 습관 차이로 추정.
어디서 - explore.py C-3
왜 이상 - 같은 컬럼인데 형식이 달라 날짜 정렬·계산이 안 됨. 
         통합 전 하나로 통일 필요.

## 4. 분석 질문 10개

형식 (3줄 엄수):
질문:
의도: (답이 나오면 누가 무엇을 바꾸는가)
필요 데이터: (테이블/컬럼 추정)

Q1.
질문: 작업지시 생성일과 착수일 사이 간격이 큰 건들은 무엇 때문에 대기했나?
\
의도: 대기 원인이 자재면 구매정책을, 설비면 부하 배분을 조정해야 함. 자재가 늦으면 → 생산이 늦고 → 납기가 늦는다
\
필요 데이터: mes_work_orders의 REG_DT/REL_DT, ERP 재고 스냅샷, BOM

Q2.
질문: (위화감 2에서) 양품수량과 출하수량이 어긋나는 21건은 특정 시기·완제품·고객에 몰려 있는가, 무작위로 흩어져 있는가?
\
의도: 몰려 있으면 시스템/공정 원인, 흩어져 있으면 입력 오류로 방향 결정
\
필요 데이터: MES 생산실적, ERP shipments, 작업지시-주문 연결

Q3.
질문 : 미출하 73건은 데이터 마지막 시점(12월)에 몰린 정상적 꼬리인가, 아니면 다른 시기에도 흩어져 있는가?
\
의도: 12월 꼬리면 자연스러운 미완료, 흩어져 있으면 누락/문제 신호
\
필요 데이터: sales_order_d(due_date), shipments

Q4.
질문 : MES 코드 중 ERP 마스터와 매핑 안 되는 코드는 몇 종이고, 불일치 유형은 몇 가지로 나뉘는가? (형식 차이 vs 미등록 코드)
\
의도: 유형별 건수를 알아야 코드 변환 규칙을 설계할 수 있음 (규칙으로 풀 것과 수동 매핑할 것의 분리)
\
필요 데이터: ERP items, MES 생산실적의 ITEM_CD 전체 목록

Q5.
질문 : 검사 Excel의 날짜 형식은 총 몇 가지이고, 각 형식은 모호함 없이 실제 날짜로 복원 가능한가? (연도 누락 형식 포함)
\
의도: 복원 불가능한 형식이 있으면 해당 데이터는 버릴지 추정할지 결정해야 하므로, 가공 규칙 설계 전에 확인 필요
\
필요 데이터: inspection Excel 전체 파일의 검사일 컬럼

Q6.
질문 : 자재 입고는 약속일(promised_date) 대비 얼마나 지켜지는가? 공급사별 지연 분포는 어떻게 다르고, 상습 지연 공급사가 있다면 언제부터 그런가?
\
의도: 상습 지연 공급사가 있으면 발주 시점을 앞당기거나 공급처를 재검토해야 하고, 특정 시점부터의 지연이면 원인 이벤트를 추적
\
필요 데이터: purchase_orders(supplier_code, order_date, promised_date), material_receipts(receipt_date), po_no로 조인

Q7.
질문 : 고객 주문의 납기(due_date)는 실제 출하(ship_date) 대비 얼마나 지켜지는가? 지연은 특정 거래처·품목군·시기에 몰려 있는가, 그리고 언제부터 나빠졌는가?
\
의도: 지연이 몰린 단면과 시작 시점을 알아야 원인 후보를 좁힐 수 있음. 특히 자재 입고 지연(Q6)과 시기·품목이 겹치는지 확인해봐야 함. "자재 지연 -> 납기 지연" 인과 가설을 검증.
\
필요 데이터: sales_order_d(due_date, item_code), shipments(ship_date), customers, items — order_no + line_no로 조인

Q8.
질문 : 8개 라인의 사이클타임(CYCLE_TM)은 서로, 시간에 따라 어떻게 다른가? 특정 라인이 느려지는 구간이 있다면 언제부터이고, 같은 품목 기준으로 비교해도 그런가?
\
의도: 라인 간·시점 간 차이가 확인되면 해당 설비의 점검/부하 재배분 대상이 됨. "같은 품목 기준" 비교로 품목 구성 차이 효과를 배제
\
필요 데이터: MES 생산실적(EQP_CD, ITEM_CD, WORK_DT, CYCLE_TM)

Q9. 
질문 : 불량률(DEF_QTY/PROD_QTY)은 라인별·시기별로 어떻게 다른가? 사이클타임이 나빠지는 구간과 불량률이 오르는 구간이 겹치는가?
\
의도: 두 지표가 같이 움직이면 설비 상태 악화라는 공통 원인을 의심, 따로 움직이면 별개 문제로 분리해서 접근
\
필요 데이터: MES 생산실적(EQP_CD, WORK_DT, PROD_QTY, DEF_QTY, CYCLE_TM)

Q10.
질문 : 재고가 0(또는 그 근처)까지 떨어진 자재는 어떤 것들이고, 언제, 얼마나 자주 그랬는가? 그 시점은 자재 대기로 멈춘 작업지시(Q1)와 겹치는가?
\
의도: 바닥난 자재가 있고 그때 생산이 멈췄다면 안전재고·발주 정책이 공급 지연을 못 버틴다는 뜻 — 발주점을 올리거나 발주를 앞당겨야 함. 공급사 지연(Q6)과 시기가 겹치는지도 확인해 사슬로 연결.
\
필요 데이터: inventory_daily(snap_date, material_code, qty_on_hand), MES 작업지시(REG_DT/REL_DT), BOM
